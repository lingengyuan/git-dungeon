# 像素游戏体验问题清单

> 日期：2026-05-07
> 范围：Pixel mode 在 Phase 0-12 完成后的整体审查。
> 依据：真实窗口试玩 `--pixel --seed 42 --lang zh_CN`，以及当前 screen / widget / text / asset / audio 相关代码与资源检查。
> 目标：让 Pixel mode 从“像素风界面原型”继续推进到“可以被当作像素游戏来玩的版本”。
> Phase 18 收口记录：见 `plans/pixel-phase18-final-playtest.md`，包含截图包、最终验证命令、P0/P1/P2 关闭与延期清单。

## 审查标准

参考标准不是“能不能运行”，而是接近《星露谷物语》这类成熟像素游戏的基础体验：

- 玩家一眼能看懂自己在哪里。
- 哪些东西能互动、哪些东西危险、哪些东西已经领取或锁住，要靠画面先说明。
- 文本是辅助，不应该承担全部解释。
- 中文模式必须像一个完整中文游戏，而不是中文、英文、调试标签混在一起。
- 音乐、美术、交互反馈要形成统一气质，而不是只把现有 CLI 结果套进像素窗口。

严重程度：

- **P0**：会阻止它被看作一个像素游戏，必须优先修。
- **P1**：继续加玩法前应该先修，否则后续功能会堆在不稳的体验上。
- **P2**：强烈影响质感、清晰度或长期维护。
- **P3**：后续路线图，可以在核心体验稳定后做。

## 真实试玩证据

本轮真实启动窗口并操作了标题页、地牢移动、奖励领取流程，确认：

- 标题页可以进入地牢。
- 地牢里可以移动到相邻房间。
- 奖励房间可以领取，领取后数值变化可见。

同时暴露出一批体验问题：

- 默认窗口过大，底部操作区靠近系统 Dock，整体显得拥挤。
- 标题页、地牢 HUD、底部提示都有文字压边或互相抢空间的问题。
- 中文模式里仍有 `Enter`、`WASD`、`HP`、`MP`、`ATK`、`Gold`、`audio: chapter` 这类英文或调试式内容。
- 地牢仍像路线节点图，不像可探索的像素房间。
- 奖励领取主要靠顶部小字和数值变化反馈，没有宝箱打开、拾取动画、音效对应反馈。
- `Esc/Q` 在运行中直接退出，没有暂停菜单或确认。

## P0 问题

### PX-001：默认窗口尺寸不够安全

- **区域**：全局窗口和布局。
- **现象**：真实窗口试玩时，游戏内容几乎铺满一个大窗口，底部操作区靠近 Dock，看起来像快被系统界面挡住。
- **问题**：固定 1280x720 放大 320x180 画布虽然能跑，但不像一个照顾桌面环境的正式游戏。
- **期望**：首次启动时窗口居中、完整可见，重要按钮和文字不靠近系统边缘。
- **整改方向**：增加桌面安全尺寸；保留 1280x720，但默认可用更小的窗口；给标题页、地牢、战斗增加布局截图检查。

### PX-002：核心屏幕存在文字压边和拥挤

- **区域**：标题页、地牢 HUD、底部控制栏。
- **现象**：标题状态文字和开始/设置区域距离过近；地牢顶部提示贴着面板边；底部提示尺寸大且靠边。
- **问题**：当前布局主要靠坐标摆放，没有统一预留标题区、状态区、世界区、操作区。
- **期望**：标题、状态、按钮、提示、边框之间都有清楚间距。
- **整改方向**：先抽出共享屏幕布局规则，再继续扩展新画面。

### PX-003：中文模式不是完整中文体验

- **区域**：所有玩家可见界面。
- **现象**：中文界面混有 `HP`、`MP`、`ATK`、`Gold`、`Enter`、`WASD`、`audio: chapter`、`+4 HP +15 Gold`。
- **问题**：这些内容让游戏像调试工具，不像中文成品。
- **期望**：中文模式里控制、属性、奖励、房间名、音频状态都统一成玩家语言。
- **整改方向**：建立一份玩家可见词汇表；统一属性、控制、奖励、钥匙、宝箱、陷阱、音乐状态的中文表达；普通界面隐藏音频内部槽位。

### PX-004：地牢还是节点图，不是像素世界

- **区域**：地牢探索。
- **现象**：画面是深色网格、图标和细连线，没有墙、地砖、门、通道、物件、光源。
- **问题**：玩家看到的是路线图，而不是进入了一个可探索的地牢。
- **期望**：地牢至少要像一个小型 tile world：地面、墙、门、通路、陷阱、宝箱、锁门、出口。
- **整改方向**：保留现有路线逻辑，但把视觉表现替换成房间和地砖系统。

### PX-005：房间类型不够自解释

- **区域**：地牢主线房间和分支房间。
- **现象**：奖励房间复用类似商店的图标；钥匙、宝库、补给、陷阱的区分不明显。
- **问题**：玩家需要读文字才能知道房间含义。
- **期望**：补给、钥匙、宝库、陷阱、战斗、事件、休息、商店、精英、Boss 都应该能靠外形和颜色先识别。
- **整改方向**：新增独立图标或 sprite；不要继续复用 `node_shop` 表达所有奖励。

### PX-006：当前房间、玩家位置、已访问、锁定、奖励状态互相抢视觉

- **区域**：地牢房间状态。
- **现象**：黄、绿、红、灰边框同时承担多种含义；玩家 sprite 会盖住房间图标；红色既可能表示危险，也可能像不可走。
- **问题**：状态含义太多，但表现手段太少。
- **期望**：玩家位置、当前可行动、危险、锁定、已领取分别有稳定视觉规则。
- **整改方向**：建立房间状态矩阵；玩家标记和房间边框分离；锁用锁图标，危险用陷阱图，已访问用变暗或打开状态。

### PX-007：玩家进入地牢后的第一个目标不清楚

- **区域**：第一次进入地牢。
- **现象**：提示写“按 Enter 进入”，但画面同时出现多个分支、陷阱和高亮节点。移动后又出现“移动到当前房间”。
- **问题**：玩家不知道现在应该进主线、拿奖励、避开陷阱，还是只是移动光标。
- **期望**：第一屏要明确告诉玩家“你现在站在哪里、能做什么、下一步是什么”。
- **整改方向**：加入首轮短提示或上下文动作条，让当前可执行的动作唯一且清楚。

### PX-008：关键反馈过度依赖小字

- **区域**：奖励领取、陷阱触发、锁门、战斗胜利。
- **现象**：领取补给后主要是顶部文字和属性数字变化，没有宝箱打开、拾取闪光、房间状态变化。
- **问题**：像素游戏应该让“发生了什么”直接出现在场景里。
- **期望**：领取奖励时有打开状态、短动画、近场浮字和音效配合。
- **整改方向**：为补给、钥匙、宝库、陷阱增加状态 sprite 和短动画。

### PX-009：`Esc/Q` 直接退出运行

- **区域**：运行中的输入行为。
- **现象**：地牢中 `Esc/Q` 会直接退出。
- **问题**：正式游戏不应该让常用按键无确认结束当前运行。
- **期望**：`Esc` 打开暂停菜单，里面有继续、设置、退出运行；退出需要确认。
- **整改方向**：新增运行中暂停菜单，并接入地牢、战斗、商店、休息、事件屏幕。

### PX-010：缺少视觉回归检查

- **区域**：测试策略。
- **现象**：逻辑测试和 parity 测试能证明结果一致，但挡不住文字压边、窗口拥挤、语言混杂。
- **问题**：最影响用户第一眼感受的问题没有自动保护。
- **期望**：核心屏幕至少有中英文截图或像素边界检查。
- **整改方向**：给标题、地牢、战斗、商店、休息、事件、设置增加 headless 渲染检查。

## P1 问题

### PX-011：标题页像菜单原型，不像游戏标题场景

- **区域**：标题页。
- **现象**：标题页主要是大字、边框、两个 sprite，没有环境、动画、logo 资产或明确氛围。
- **问题**：成熟像素游戏会用标题页建立世界观和情绪。
- **期望**：标题页能看到地牢入口、commit 碎片、火把、门、角色或专属标题图。
- **整改方向**：生成或绘制正式标题背景/标题 banner，菜单只作为下方控制区。

### PX-012：美术风格过于通用

- **区域**：整体资产。
- **现象**：当前主要依赖 Kenney Tiny Dungeon 和暗色面板；项目专属资产还没有形成体系。
- **问题**：画面不像“Git Dungeon”，更像通用地牢素材测试。
- **期望**：角色、敌人、Boss、道具能表达开发、仓库、分支、冲突、CI、发布等主题。
- **整改方向**：完成 gpt-image-2 资产流水线；每个资产保留 prompt、原图、处理步骤、授权和用途说明。

### PX-013：Boss 没有身份感

- **区域**：战斗和 Boss。
- **现象**：战斗主要使用默认敌人 sprite，生成 Boss 的计划还没有真正接入。
- **问题**：Boss 战如果看起来和普通敌人差不多，就没有里程碑感。
- **期望**：每类 Boss 有独立外观、颜色、入场提示和战斗氛围。
- **整改方向**：接入 Boss sprite manifest，先做 contact sheet 审查，再进入游戏资源。

### PX-014：战斗视觉反馈太薄

- **区域**：战斗。
- **现象**：攻击、防御、技能主要靠按钮和少量闪烁表达。
- **问题**：玩家感觉不到攻击节奏、受击、格挡、释放技能和击败敌人。
- **期望**：攻击、格挡、技能、敌人受击、敌人失败都有清楚短动画。
- **整改方向**：加入玩家前冲/挥击、敌人震动、护盾效果、技能特效、失败淡出和奖励落下。

### PX-015：界面会暴露内部 ID 或原始字段

- **区域**：事件、商店、战斗。
- **现象**：事件会显示 event/choice 风格 ID；商店详情有 `cost hp atk mp maxhp` 这类原始字段。
- **问题**：玩家看到的是数据结构，不是游戏文案。
- **期望**：所有可见名称都应是玩家语言。
- **整改方向**：为 Pixel UI 建一层显示文案格式化，不直接显示内部字段名。

### PX-016：UI 控件不像像素游戏控件

- **区域**：按钮、设置、商店。
- **现象**：按钮是普通矩形加文字；音量用加减；设置提示像快捷键说明。
- **问题**：整体像工具界面，不像游戏界面。
- **期望**：按钮、滑杆、开关、弹窗、提示都使用统一像素 UI kit。
- **整改方向**：建立小型 pixel UI kit：按钮、禁用态、选中态、滑杆、弹窗、toast、提示框。

### PX-017：鼠标支持不符合画面预期

- **区域**：地牢导航。
- **现象**：鼠标能点 Enter 按钮，但不能点击房间移动或查看房间。
- **问题**：画面像地图，玩家自然会想点击地图。
- **期望**：点击相邻房间可移动；点击锁门、陷阱、奖励可查看提示。
- **整改方向**：给地牢 tile 增加点击命中和查看/移动行为。

### PX-018：底部帮助不够上下文相关

- **区域**：全局操作提示。
- **现象**：底部经常显示通用移动和 Enter 提示，不随奖励、陷阱、锁门、当前房间变化。
- **问题**：玩家不知道按 Enter 会发生什么。
- **期望**：动作条显示“领取补给”“开始战斗”“需要铁钥匙”“触发陷阱”等明确动作。
- **整改方向**：建立统一 action bar：主动作、次动作、警告、不可用原因。

### PX-019：音频状态是调试语言

- **区域**：HUD 和音频。
- **现象**：界面显示 `音频: chapter`。
- **问题**：玩家不需要知道内部 BGM 槽位。
- **期望**：普通界面不显示音频槽；只在静音、错误或设置页展示必要信息。
- **整改方向**：隐藏正常音频状态；错误时显示可理解的提示。

### PX-020：加载可能阻塞窗口

- **区域**：加载页。
- **现象**：仓库加载在 UI 主流程里进行，小仓库没问题，大仓库可能卡住窗口。
- **问题**：窗口卡死会像程序坏了。
- **期望**：加载时窗口继续刷新，并给出进度或阶段提示。
- **整改方向**：改成后台加载或分阶段加载，至少显示“读取提交/生成路线/准备资源”等阶段。

### PX-021：陷阱风险没有提前说明

- **区域**：陷阱。
- **现象**：陷阱只是红色框，伤害只在触发后知道。
- **问题**：玩家无法做有信息的选择。
- **期望**：接近或选中陷阱时看到危险提示和预计损失。
- **整改方向**：增加陷阱 sprite、悬停/选中提示、触发前警告。

### PX-022：钥匙和宝库状态没有进入 HUD

- **区域**：钥匙/宝库分支。
- **现象**：系统能记录已获得钥匙，但地牢 HUD 不显示钥匙，也不明显说明宝库需求。
- **问题**：玩家拿到钥匙后不一定知道能开哪里。
- **期望**：HUD 有小钥匙槽；锁门或宝库显示需要哪把钥匙。
- **整改方向**：增加小型关键物品区，并把锁门需求可视化。

### PX-023：世界缺少待机动画和生命感

- **区域**：所有屏幕。
- **现象**：真实试玩中，除了切换画面和少量战斗闪烁，大部分画面是静态的。
- **问题**：像素游戏需要小幅运动来保持生命感。
- **期望**：角色待机、火把、传送门、按钮选中、陷阱闪烁、奖励闪光都有轻量动画。
- **整改方向**：引入简单动画调度和 sprite 帧支持。

### PX-024：运行中不能安全进入设置

- **区域**：运行流程。
- **现象**：设置只在标题页可进；运行中按 Esc 是退出。
- **问题**：玩家无法在运行中调音量、窗口、语言，也不能安全暂停。
- **期望**：暂停菜单可以继续、设置、返回标题、退出运行。
- **整改方向**：做一个共享暂停层，接入所有运行中屏幕。

### PX-025：路线缺少推进感

- **区域**：地牢布局。
- **现象**：房间排列像紧凑节点线，分支很抽象。
- **问题**：玩家感觉不到入口、深入、危险区、宝库、Boss 门的进展。
- **期望**：地图有空间推进：入口、走廊、岔路、奖励房、危险房、Boss 门。
- **整改方向**：在不改路线逻辑的前提下重做空间排布和章节主题。

## P2 问题

### PX-026：调色板过于单一

- **区域**：视觉设计。
- **现象**：大多数屏幕都是深色背景、琥珀边框/文字、灰字、红绿状态。
- **问题**：短期可读，但长时间玩会显得单调。
- **期望**：标题、地牢、战斗、Boss、休息、商店、事件有不同强调色，但仍统一。
- **整改方向**：定义章节和屏幕级调色板。

### PX-027：视觉层级倒置

- **区域**：整体层级。
- **现象**：状态和标题很大，真正的房间、角色、交互物反而小。
- **问题**：玩家注意力被 UI 抢走，而不是看游戏世界。
- **期望**：世界画面是主角，文本只在需要时解释。
- **整改方向**：缩小常驻 HUD，放大或强化房间和对象。

### PX-028：门和路径不够像可通行空间

- **区域**：地牢连接。
- **现象**：房间之间是细线。
- **问题**：细线更像图表连线，不像门或走廊。
- **期望**：通路应表现为地砖、门洞、门框或走廊。
- **整改方向**：用 corridor tile / door tile 替换线段。

### PX-029：奖励领取后的状态太弱

- **区域**：奖励房间。
- **现象**：领取后主要是提示和边框变化。
- **问题**：玩家容易忘记哪个奖励已经拿过。
- **期望**：已打开宝箱、空补给箱、暗淡房间状态持续显示。
- **整改方向**：增加 claimed sprite 和 room dim 状态。

### PX-030：战斗底部区域太挤

- **区域**：战斗 UI。
- **现象**：动作按钮和结果文字共用较小底部区域。
- **问题**：信息容易被压缩，按钮也显得局促。
- **期望**：动作、战斗日志、提示各有固定位置。
- **整改方向**：拆分战斗布局：角色区、敌人区、动作条、最近日志。

### PX-031：商店信息不够玩家友好

- **区域**：商店。
- **现象**：物品效果显示成 `cost`、`hp`、`atk`、`mp`、`maxhp`。
- **问题**：像在看数据表。
- **期望**：商品有图标、名称、价格、效果摘要、买不起的原因。
- **整改方向**：做商品卡片和购买反馈。

### PX-032：事件选择缺少主题和后果预览

- **区域**：事件。
- **现象**：事件页偏向显示 ID 和效果。
- **问题**：不像一次冒险事件，更像调试选择。
- **期望**：事件有标题、短文本、选择、预计后果。
- **整改方向**：补玩家文案层；可结合 AI/模板生成，但必须缓存和清洗。

### PX-033：休息点不像一个地点

- **区域**：休息。
- **现象**：休息页主要是选项面板。
- **问题**：缺少营地、神龛、安全屋这类场景感。
- **期望**：休息点有背景/物件，回血和恢复有可见效果。
- **整改方向**：增加休息场景 sprite 和短效果动画。

### PX-034：设置页过度展示文件路径

- **区域**：设置。
- **现象**：设置页靠近标题处显示配置文件路径。
- **问题**：这对开发有用，但对玩家是噪音。
- **期望**：普通玩家只看到设置项；路径只在调试或错误详情里出现。
- **整改方向**：隐藏路径，保留在 debug/error 模式。

### PX-035：窗口模式切换反馈不即时

- **区域**：设置。
- **现象**：设置里提示需要重启。
- **问题**：作为简单像素游戏，这会显得没做完。
- **期望**：窗口/全屏能立即应用，或明确给出确认流程。
- **整改方向**：实现即时重设窗口；如果确实需要重启，要给出清楚确认。

### PX-036：没有手柄支持

- **区域**：输入。
- **现象**：目前是键盘加部分鼠标。
- **问题**：像素 RPG 玩家常预期手柄可玩。
- **期望**：方向键/摇杆、确认、取消、菜单基础可用。
- **整改方向**：核心键鼠清楚后再加可选手柄映射。

### PX-037：可访问性选项不足

- **区域**：可访问性。
- **现象**：没有文字大小、高对比度、减少动画、色盲友好选项。
- **问题**：当前大量依赖颜色边框和小字。
- **期望**：至少支持文字大小和高对比模式。
- **整改方向**：布局稳定后增加可访问性设置。

### PX-038：音乐资源合法，但还没有统一听感

- **区域**：音乐和音频方向。
- **现象**：BGM 来源目前是合法的 CC0 资源，但来自不同包/作者，按 slot 使用。
- **问题**：版权安全不等于听感统一。
- **期望**：标题、章节、Boss、失败音乐有统一气质、音量、循环处理。
- **整改方向**：做完整试听；统一响度；剪循环点；记录每首曲子的用途和选择原因。

### PX-039：缺少玩家可见的过程记录

- **区域**：运行进展。
- **现象**：奖励和事件会改变数值，但没有清楚的最近事件或章节总结。
- **问题**：玩几步后容易忘记刚发生了什么。
- **期望**：有最近获得、损失、关键事件的小日志或章节总结。
- **整改方向**：增加近期事件日志或章节完成页。

### PX-040：缺少第一次玩的引导

- **区域**：新手体验。
- **现象**：玩家直接进入符号密集的路线图。
- **问题**：默认假设玩家已经理解节点系统。
- **期望**：前 30 秒教会移动、进入房间、领奖励、识别陷阱、理解当前房间。
- **整改方向**：增加首轮提示或固定教学种子。

## P3 问题

### PX-041：Git 主题在世界里不够明显

- **区域**：世界观和识别度。
- **现象**：标题叫 Git Dungeon，但房间和敌人仍是通用地牢感。
- **问题**：项目概念比当前画面更强。
- **期望**：commit、branch、merge conflict、CI、secret、release gate 这些主题能变成房间、陷阱、敌人、道具。
- **整改方向**：建立 Git 主题视觉语言，例如分支门、commit 碎片、冲突陷阱、CI 守卫、发布关口。

### PX-042：没有 NPC、环境物或可互动小物件

- **区域**：世界丰富度。
- **现象**：房间里主要是节点图标、陷阱、奖励。
- **问题**：空间缺少生活感和叙事信息。
- **期望**：商人、引导角色、commit ghost、安全屋物件能增加个性。
- **整改方向**：核心体验稳定后加入安全房道具、商人、引导角色或环境叙事。

### PX-043：章节没有地貌/主题变化

- **区域**：章节推进。
- **现象**：不同章节视觉语言基本一致。
- **问题**：长线游玩会重复。
- **期望**：每章有小幅 palette、地砖、敌人、音乐变化。
- **整改方向**：先稳定基础房间系统，再做章节主题包。

### PX-044：Pixel mode 标题流程缺少 Continue/New Run/Daily 等游戏化入口

- **区域**：标题和开始流程。
- **现象**：标题页主要是 Start 和 Settings。
- **问题**：作为游戏前端，玩家会期待继续游戏、新运行、每日挑战、选择 seed 等入口。
- **期望**：标题流程能区分继续、新开、每日、设置。
- **整改方向**：单局闭环打磨后，再设计开始菜单和存档/继续流程。

## 建议修复顺序

1. **Phase 13A：布局和可读性急救**
   解决窗口安全尺寸、屏幕分区、文字间距、中文词汇统一、隐藏调试音频标签。

2. **Phase 13B：地牢视觉语言重做**
   明确补给、钥匙、宝库、陷阱、锁定、当前、已领取状态；加入上下文动作条。

3. **Phase 14：真正的 tile 房间表现**
   引入地砖、墙、门、走廊、打开后的宝箱、陷阱预览。

4. **Phase 15：战斗表现补强**
   接入专属敌人/Boss sprite，补攻击、防御、技能、受击、胜利反馈。

5. **Phase 16：非战斗场景打磨**
   商店、休息、事件从数据面板变成场景。

6. **Phase 17：美术和音乐方向统一**
   完成 gpt-image-2 资产流水线；做音乐试听、音量统一、循环处理和来源记录。

7. **Phase 18：视觉回归保护**
   对中英文核心屏幕和常见桌面尺寸建立截图/像素边界检查。

## “可以叫像素游戏”的最低验收线

在下面这些条件满足前，Pixel mode 更准确地说还是“像素 UI 原型”，不是完整像素游戏：

- 首次打开窗口完整可见、居中、没有内容贴近系统边缘。
- 标题、地牢、战斗、商店、休息、事件、设置在中英文下都可读。
- 地牢看起来是 tile 世界，不只是路线节点图。
- 每种房间类型都有独立视觉身份。
- 奖励、陷阱、锁门、钥匙不用读代码或文档也能理解。
- `Esc` 打开暂停菜单，而不是直接退出。
- 至少一场战斗有清楚的攻击、防御、技能和胜负反馈。
- 普通玩家界面不出现内部 ID、原始字段、音频槽位等调试内容。
- 上述关键画面有视觉检查保护，避免后续改坏。

# Pixel Game 修复计划

> 日期：2026-05-07
> 输入：`plans/pixel-game-issues.md`
> 目标：把 44 个体验问题拆成可以逐 phase 落地、验证、交接的修复路线。
>
> 2026-05-07 更新：后续修复标准已升级为 `plans/pixel-stardew-level-repair-plan.md`。Phase 13R review findings 已完成；下一步从 Phase 14A 统一 UI kit 和玩家语言层开始，再做 gpt-image-2 素材流水线和场景化重做。

## 修复原则

- 先修玩家第一眼和第一分钟会遇到的问题，再修大规模美术、音乐和章节内容。
- 每个 phase 都必须能独立验证，完成后写 handoff。
- 路线和战斗结算仍以现有核心逻辑为准；Pixel 层只做表现、输入和可选支线体验。
- 不用静默 fallback 掩盖缺资源、缺文案或设置错误。

## Phase 13：可读性和安全交互

**目标**：先让现有玩法“看得清、退得安全、知道该按什么”。

覆盖问题：

- PX-001 默认窗口尺寸不够安全
- PX-002 核心屏幕文字压边和拥挤
- PX-003 中文模式混杂英文/调试语言
- PX-005 房间类型不够自解释
- PX-006 房间状态互相抢视觉
- PX-007 第一个目标不清楚
- PX-008 关键反馈过度依赖小字
- PX-009 `Esc/Q` 直接退出
- PX-017 鼠标不能直接操作地图
- PX-018 底部帮助不够上下文相关
- PX-019 音频状态是调试语言
- PX-022 钥匙和宝库状态没有进入 HUD
- PX-034 设置页过度展示文件路径

交付：

- 默认窗口改成更安全的桌面尺寸。
- 普通游戏界面隐藏正常音频槽位，只保留静音或错误提示。
- 统一中文玩家词汇：生命、魔力、攻击、金币、经验、确认键、铁钥匙等。
- 地牢动作条显示当前主动作，例如领取补给、进入当前房间、需要铁钥匙。
- 地牢奖励、钥匙、宝库、陷阱、锁定、已领取用不同图形/状态表现。
- 点击相邻房间可以移动；点击非相邻房间显示可理解提示。
- 运行中 `Esc/Q` 打开暂停确认，不再直接退出。
- 设置页不再默认展示配置文件路径。
- 自动测试覆盖暂停确认、中文文案、音频标签隐藏、鼠标点击移动。

## Phase 14：统一 UI、素材流水线和 tile 房间表现

**目标**：先统一 Pixel UI 和玩家语言，打通 gpt-image-2 地牢素材流水线，再把地牢从“路线节点图”推进到“小型像素房间”。

覆盖问题：

- PX-004 地牢还是节点图
- PX-021 陷阱风险没有提前说明
- PX-025 路线缺少推进感
- PX-028 门和路径不像可通行空间
- PX-029 奖励领取后状态太弱
- PX-040 缺少第一次玩的引导

交付：

- 统一 UI kit 与玩家语言 formatter，后续 screen 不再各自拼玩家文案。
- gpt-image-2 地牢基础素材先完成 prompt、原图、后处理、contact sheet、asset card、manifest 校验。
- 地砖、墙边、门、走廊替换抽象连线。
- 当前房间、主线方向、支线奖励、危险区形成稳定空间布局。
- 陷阱选中/靠近时显示预计损失。
- 奖励领取后保留打开后的宝箱或空房间状态。
- 第一轮进入地牢时有短引导或固定教学提示。

## Phase 15：战斗表现补强

**目标**：让战斗像像素 RPG 战斗，而不是按钮面板。

覆盖问题：

- PX-013 Boss 没有身份感
- PX-014 战斗视觉反馈太薄
- PX-030 战斗底部区域太挤

交付：

- 战斗区、动作区、战斗日志分离。
- 攻击、防御、技能、受击、胜利各有独立短反馈。
- Boss 使用独立 sprite、入场提示和颜色气氛。

## Phase 16：非战斗场景和玩家文案

**目标**：把商店、事件、休息从数据面板改成游戏场景。

覆盖问题：

- PX-015 暴露内部 ID 或原始字段
- PX-016 UI 控件不像像素游戏控件
- PX-024 运行中不能安全进入设置
- PX-031 商店信息不友好
- PX-032 事件选择缺少主题和后果预览
- PX-033 休息点不像一个地点
- PX-035 窗口模式切换反馈不即时
- PX-039 缺少玩家可见过程记录
- PX-044 缺少 Continue/New Run/Daily 入口

交付：

- 商店商品卡、休息场景、事件叙事文本和后果预览。
- 统一 pixel UI kit：按钮、禁用态、滑杆、弹窗、toast。
- 暂停菜单接入设置页。
- 标题入口区分新运行、继续、每日挑战和设置。

## Phase 17：美术、动画和音乐方向统一

**目标**：形成真正的 Git Dungeon 视觉和听觉身份。

覆盖问题：

- PX-011 标题页不像游戏标题场景
- PX-012 美术风格过于通用
- PX-023 世界缺少待机动画
- PX-026 调色板过于单一
- PX-027 视觉层级倒置
- PX-038 音乐合法但听感不统一
- PX-041 Git 主题不够明显
- PX-042 没有 NPC、环境物或小物件
- PX-043 章节没有主题变化

交付：

- gpt-image-2 资产流水线完成：prompt、原图、后处理、asset card、manifest、contact sheet。
- 标题场景、玩家、Boss、关键道具、Git 主题物件进入游戏。
- 待机、火把、陷阱、奖励、按钮等轻量动画。
- 音乐完整试听、响度统一、循环处理、用途记录。

## Phase 18：视觉回归保护

**目标**：防止后续改动再次把画面改到不可读。

覆盖问题：

- PX-010 缺少视觉回归检查
- PX-036 没有手柄支持
- PX-037 可访问性选项不足

交付：

- 中英文标题、地牢、战斗、商店、休息、事件、设置截图或像素边界检查。
- 常见桌面尺寸覆盖。
- 高对比和文字大小选项。
- 核心体验稳定后补手柄输入映射。

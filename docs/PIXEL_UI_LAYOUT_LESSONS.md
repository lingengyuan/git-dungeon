# Pixel UI Layout Lessons

## 背景

Phase 14A 之后，战斗、地牢、商店、标题页连续暴露了多处像素 UI 问题。表面看是“字太大”“线条碰到文字”“红块压住血条”，本质都是同一类错误：在 320x180 的固定画布里，没有先划清每个元素的占位区域，就直接用绝对坐标叠上去。

这份文档记录已经遇到的问题、修复原则和后续检查清单。以后做像素界面改动，先读这里，再改屏幕。

## 已遇到的问题

1. 中文字体缺字，部分文字只显示方块。
2. 中文字体实际高度偏大，文字看起来快要顶出按钮或面板。
3. 地牢页金币文字和状态栏边框贴在一起。
4. 战斗页角色名压到血条。
5. 战斗页生命、魔力和动作按钮发生重叠。
6. 玩家生命文字压到绿色玩家块。
7. 敌人红色块压到红色血条。
8. 敌人生命、攻击没有和玩家生命、魔力对齐。
9. 标题页敌人块压到 `GIT DUNGEON` 标题。
10. 商店页底部“跳过”按钮被底部提示栏盖住。

## 根因

- 只看单个元素，没有先定义整屏网格。
- 文本、图块、按钮共享同一块区域，没有留出安全间距。
- 依赖“看起来差不多”的坐标，而不是用常量和测试固定关系。
- 只跑启动 smoke，没有生成截图做人工复查。
- 中文和英文共用同样字号，但中文字体的视觉高度更高。
- 绘制顺序没有被当成布局规则，导致后画的底栏盖住先画的按钮。

## 规则

1. 文字不能直接贴图块，至少留 6px；标题级文字至少留 8px。
2. 文字不能直接贴边框，底部至少留 3px。
3. 同类状态必须共享行基准：玩家生命对敌人生命，玩家魔力对敌人攻击。
4. 动作按钮和底部提示栏之间至少留 3px。
5. 如果按钮放在底部提示栏上方，必须先画提示栏，再画按钮。
6. 中英文都要走 `draw_fit`，除非文字长度绝对固定且已有截图验证。
7. 新增或移动 UI 元素时，坐标要变成命名常量，关键关系要写测试。
8. 每次视觉修复必须重新生成截图，不能只相信代码。

## 检查清单

- 标题是否和角色块、边框保持间距。
- 角色块是否和血条、生命、魔力、攻击保持间距。
- 状态文字是否左右对齐。
- 按钮是否被底栏、面板或其他按钮覆盖。
- 中文是否完整显示，没有方块字。
- 中文是否没有贴线、贴边、压框。
- 操作提示是否给右侧按钮预留空间。
- 绘制顺序是否符合“背景 → 面板 → 内容 → 底栏 → 浮在底栏上的按钮”。
- 截图是否覆盖标题、地牢、战斗、事件、休息、商店、设置、暂停、结算和错误页。

## 当前复查结论

2026-05-08 的深度复查新增发现并修复了两处同类问题：

- 标题页：`GIT DUNGEON` 标题和右侧敌人块区域冲突，已改为固定标题宽度并收小标题字号。
- 商店页：底部“跳过”按钮被底栏覆盖，已调整为先画底栏，再画按钮。
